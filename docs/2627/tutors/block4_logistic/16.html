Probabilities:

  • 33%
  • 99%
  • 20%
  • 0%
  • 100%

Solution

Solution 2. Eyeballing a path from the probabilities on the y axis via the logistic function to the log-odds values on the x axis, I’d say:

  • 33% probability \(\approx\) –0.5 log-odds
  • 99% probability \(\approx\) 4.5 log-odds
  • 20% probability \(\approx\) –1.5 log-odds
  • 0% probability is never met, only approached, as the logistic function goes down to \(-\infty\)
  • 100% probability is never met, only approached, as the logistic function goes up to \(\infty\)

Question 4

Try running the following three lines of code. They will fail to return sensible log-odds values.

Explain why each line fails.

qlogis(-0.5)
qlogis(1.5)
qlogis(100)

Solution

Solution 4. None of the given values are possible probabilities, so there is no way to map them to log-odds.

  • –0.5: Probabilities cannot be negative.
  • 1.5 and 100: Probabilities cannot be greater than 1.

Question 7

Can you find any input values that cause plogis() to fail?

If you do find some values, which are they, and why do they make plogis() fail?

If you cannot find any such values, why are there no values that make plogis() fail?

Solution

Solution 7. There are no values that make plogis() fail. It will take any number (even -Inf and Inf) and map it via the logistic function to a probability between 0 and 1.

This is because the input to plogis() are values on the log-odds scale. And log-odds can be any number between \(-\infty\) and \(\infty\). (This property is exactly what makes log-odds a suitable scale for linear modelling!)

Why we can’t back-transform slopes from log-odds space to probability space

Question 8

Imagine a line in log-odds space which has an intercept of 1 and a slope of 0.5. Here’s a plot of that line:

Code 
tibble(
  x = 0:5,
  logodds_success = 1 + 0.5 * x
) |>
  ggplot(aes(x = x, y = logodds_success)) +
  geom_line()

Now, imagine doing the wrong thing and back-transforming both model coefficients from log-odds space to probability space. This is a “proof by contradiction”: we’ll intentionally do the wrong thing and see how it leads us to nonsense results.

  • For the intercept, plogis(1) = 0.73.
  • For the slope, plogis(0.5) = 0.62.

The linear expression for this model is \(y = 0.73 + 0.62 \cdot x\).

What outcome value (i.e., \(y\) value) does this model predict for \(x = 0\)? for \(x = 1\)? for \(x=2\)?

Solution

Solution 8. For \(x = 0\):

\[ \begin{align} y &= 0.73 + 0.62 \cdot 0 \\ y &= 0.73 \\ \end{align} \]


For \(x = 1\):

\[ \begin{align} y &= 0.73 + 0.62 \cdot 1 \\ y &= 1.35 \\ \end{align} \]

For \(x = 2\):

\[ \begin{align} y &= 0.73 + 0.62 \cdot 2 \\ y &= 0.73 + 1.24 \\ y &= 1.97 \\ \end{align} \]

Question 9

Our goal is for the outcome values of this model to be probabilities: the probability of success.

Are the outcome values you computed in Q8 possible probabilities?

Solution

Solution 9. Probabilities can range between 0 and 1. Therefore:

  • 0.73 is a possible probability (this is the y-intercept in probability space).
  • 1.35 is not a possible probability, because it’s greater than 1.
  • 1.97 is also not a possible probability, because it’s greater than 1.

Because our model generates nonsense predictions (i.e., impossible probability values), the assumptions behind the model must be incorrect. Specifically, our assumption that a linear relationship in log-odds translates exactly to a linear relationship in probability space is not right.

Question 10

Below are two plots.

Plot A (left) shows the intentionally wrong way we tried to convert the log-odds line into probability space: by transforming each parameter individually to end up with \(y = 0.73 + 0.62 \cdot x\).

Plot B (right) shows the correct way of converting the log-odds line into probability space.

Code 
p_prob_bad <- tibble(
  x = 0:5,
  prob_success = 0.73 + 0.62 * x
) |>
  ggplot(aes(x = x, y = prob_success)) +
  geom_line() +
  ggtitle('Bad transformation to prob. space')


p_prob_good <- tibble(
  x = 0:5,
  logodds_success = 1 + 0.5 * x,
  prob_success = plogis(logodds_success)
) |>
  ggplot(aes(x = x, y = prob_success)) +
  geom_line() +
  ggtitle('Good transformation to prob. space')

p_prob_bad + p_prob_good + plot_annotation(tag_levels = 'A')

Notice that we get the same value in both plots when \(x = 0\): the probability is indeed 0.73 = plogis(1). So transforming the intercept alone is okay!

Based on these two plots, try to explain in your own words why transforming the slope directly is not a mathematically appropriate way to convert a line in log-odds space into probability space.

Solution

Solution 10. You might observe that the bad transformation gives us impossible probabilities (> 1), while the good transformation only gives us reasonable probabilities (between 0 and 1).

That’s absolutely true, and that observation is the goal of this lab.

If you want to go a bit deeper, you might also look at Plot B and observe that the good line isn’t a straight line at all: in fact, the slope is constantly changing as \(x\) increases. This is because the logistic function has to squish the straight line down so that it fits in between 0 and 1.

Because the line in probability space is not straight, there’s no single slope value that applies to the whole line. Consequently, it doesn’t make sense to transform the slope from log-odds space to probability space and expect the resulting straight line to accurately describe the probability of success.
