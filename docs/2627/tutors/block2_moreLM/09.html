Research question (RQ): Is there an association between wellbeing and time spent outdoors, when controlling for the effect that social interaction has on wellbeing?

Data dictionary:

variable description
age Age in years of respondent
outdoor_time Self report estimated number of hours per week spent outdoors
social_int Self report estimated number of social interactions per week (both online and in-person)
routine Binary 1=Yes/0=No response to the question 'Do you follow a daily routine throughout the week?'
wellbeing Warwick-Edinburgh Mental Wellbeing Scale (WEMWBS), a self-report measure of mental health and wellbeing. The scale is scored by summing responses to each item, with items answered on a 1 to 5 Likert scale. The minimum scale score is 14 and the maximum is 70
location Location of primary residence (City, Suburb, Rural)
steps_k Average weekly number of steps in thousands (as given by activity tracker if available)
NoteMore detail about this dataset

From the Edinburgh & Lothians, 100 city/suburb residences and 100 rural residences were chosen at random and contacted to participate in the study. The Warwick-Edinburgh Mental Wellbeing Scale (WEMWBS) was used to measure mental health and wellbeing.

Participants filled out a questionnaire including items concerning: estimated average number of hours spent outdoors each week, estimated average number of social interactions each week (whether on-line or in-person), whether a daily routine is followed (yes/no). For those respondents who had an activity tracker app or smart watch, they were asked to provide their average weekly number of steps.

Bootstrapping

Question 1

Read in the data from https://uoepsy.github.io/data/wellbeing_rural.csv and store it in a variable named mwdata (“mw” stands for “mental wellbeing”).

Use the function lm() to fit the linear model represented by the mathematical model formulation below, and name the result m1.

\[ \text{wellbeing} = \beta_0 + (\beta_1 \cdot \text{outdoor\_time}) + (\beta_2 \cdot \text{social\_int}) + \epsilon \]

mwdata <- read_csv("https://uoepsy.github.io/data/wellbeing_rural.csv")
m1    <- lm(wellbeing ~ outdoor_time + social_int, data = mwdata)

Question 2

Here’s a plot from last week’s lab that illustrates a problem with this model’s residuals:

plot(m1, which = 1)

Make an attempt to write down from memory (a) what assumption this plot is checking, (b) what each point represents, and (c) the problem with the model’s results that this plot illustrates.

After you’ve made a good-faith effort to respond to each point from memory, you can revisit last week’s lab/notes to complete your responses.

  1. This is a residuals-vs-fitted plot, and it’s used to visually assess the assumption of equal variance of errors.

  2. Each point represents one observation from our dataset: specifically, its residual (y axis) and its model-fitted/model-predicted outcome value (x axis).

  3. The problem is that these data points appear to build a fan/funnel/cone shape, where the vertical spread is much smaller on one side of the axis compared to the other. This indicates unequal variance, and it means we can’t trust the model’s estimates of variability, because they won’t apply equally to the whole dataset.

Question 3

Because the assumption shown in the plot above is not met, we must bootstrap our linear model’s estimates.

Use the function Boot() to draw 1000 bootstrapped samples. Name the output m1_boot.

TipCode hint

Replace the ... in the code below with the appropriate values.

... <- Boot(..., R = ...)

🗂️ See Bootstrapping > In R flash card.

m1_boot <- Boot(m1, R = 1000)

Question 4

Copy and run the following line of code.

summary(m1_boot)

Identify (a) the column of the output that represents the coefficent estimate, and (b) the column of the output that represents the standard error (i.e., the standard deviation of the sampling distribution).

summary(m1_boot)

Number of bootstrap replications R = 1000 
             original   bootBias   bootSE  bootMed
(Intercept)  28.62018 -0.0684978 1.469522 28.56650
outdoor_time  0.19909  0.0023904 0.052382  0.20086
social_int    0.33488  0.0019853 0.094009  0.33647

  1. original is the column containing the coefficient estimate.

  2. bootSE is the column containing the bootstrapped standard error.

Question 5

Use the function confint() to get the 95% CIs for each coefficient estimate.

For each coefficient, can we reject the H0 that the coefficient is equal to zero?

confint(m1_boot)
Bootstrap bca confidence intervals

                   2.5 %     97.5 %
(Intercept)  25.57997319 31.2263910
outdoor_time  0.09600595  0.3016871
social_int    0.16235434  0.5258296

None of these 95% CIs contain zero, so for all three coefficients, we can reject the H0 at \(\alpha\) = .05.

Develop study materials/strategies

Question 6

If you reach this point and still have a bit of time to dedicate to DAPR this week, we’d recommend that you start thinking about study/revision materials and strategies.

Yes, the time of year when you’ll write your reports and exams is still far away, but your learning will develop best through little bits of continuous effort. So the more prep you do now, the more your understanding will grow over time.

Here are some of Elizabeth’s favourite study methods (note that it’s the process here that will help you learn, much more than the end product):

  1. Mind mapping: Create a visual representation of how concepts are connected. Why not start one now and keep updating it as the course goes on?
  2. Flash cards (and other active recall methods): Testing yourself using active recall now will train your brain in retrieving key information, so that it’s there when you need it later.
    • If you like using physical flash cards, try the Leitner system, a system that adds in spaced repetition to help build your long-term memory.
  3. The Feynman technique: Teach or explain an idea as simply as you can. Wherever you’re not sure what to say or how to explain something, brush up on those areas and try again.
